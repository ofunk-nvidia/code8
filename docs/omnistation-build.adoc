= Omnistation Build Flow
:toc:
:sectnums:

== Purpose

Builds and packaging checks should run on Omnistation so the repository has a reproducible remote validation path.

== Expected Remote Setup

The Omnistation host needs:

* Git
* Node.js 18.19.1 or newer for the current MVP build
* npm 9.2.0 or newer
* Network access to npm registry

The package tool `@vscode/vsce` is pinned in `package.json` so packaging stays compatible with the current Omnistation Node runtime. Node 20+ remains preferred for future release automation.

== Build Commands

[source,bash]
----
git clone https://github.com/ofunk-nvidia/code8.git
cd code8
npm install
npm run build
----

== Package Command

[source,bash]
----
npm run package
----

The package command creates a `.vsix` extension package.

== CI Direction

The next step is to add a GitHub Actions workflow once the repository policy is clear. Omnistation remains the preferred manual build and release validation environment.
