= Code8 Architecture
:toc:
:sectnums:

== Goal

Code8 starts as a VS Code extension rather than a full editor fork. That keeps the first build small enough to test while still delivering the important user experience: a coding agent embedded in the editor with workspace tools.

== Components

[cols="1,2",options="header"]
|===
| Component | Responsibility

| `src/extension.ts`
| Extension activation, command registration, and SecretStorage wiring.

| `src/webview/*`
| Activity Bar chat view, message bridge, and VS Code-themed UI.

| `src/ngcProvider.ts`
| OpenAI-compatible streaming chat-completions client for NVIDIA NGC / NIM.

| `src/nimCatalog.ts`
| Dynamic NIM model discovery from the provider `/models` endpoint with official NVIDIA docs fallback.

| `src/agent/*`
| Prompt contract, action parsing, and model/tool iteration loop.

| `src/tools/toolRegistry.ts`
| Tool metadata, Plan/Act mode gates, and dispatch boundary.

| `src/tools/workspaceTools.ts`
| Workspace file operations, diff review, diagnostics, and captured command execution.
|===

== Agent Loop

[mermaid]
....
flowchart TD
  User[User prompt in Code8 view] --> Agent[Agent prompt + conversation state]
  Agent --> NGC[NVIDIA NGC / NIM chat completions]
  NGC --> Parse[Parse JSON action]
  Parse -->|read/list| Tools[Workspace tools]
  Parse -->|diagnostics/current file| Editor[VS Code editor state]
  Parse -->|write/command| Approval[VS Code approval prompt]
  Approval --> Tools
  Editor --> Result
  Tools --> Result[Tool result appended to context]
  Result --> Agent
  Parse -->|respond| Done[Final response]
....

== Action Protocol

The model is instructed to emit exactly one JSON object per step. This avoids assuming model-side tool calling support and works with OpenAI-compatible chat endpoints.

[source,json]
----
{"action":"read_file","path":"src/extension.ts","say":"I need the activation code before editing."}
----

Supported actions:

* `respond`
* `list_files`
* `read_file`
* `read_active_file`
* `list_diagnostics`
* `write_file`
* `replace_in_file`
* `run_command`

== Session Model

The webview keeps an agent session alive across multiple user messages. This gives the agent conversational continuity for a task, while `New` resets the session when the user wants a clean context. `Stop` aborts the in-flight model request through `AbortController`.

Changing the selected NIM model resets the session because model-specific behavior and context should not silently mix.

== Plan And Act

Code8 starts in Plan mode. In Plan mode, write and command tools are blocked by the runtime even if a model asks for them. This is intentionally redundant with the prompt instruction; the runtime gate is the safety boundary.

Act mode enables mutating tools, but each write or command still goes through a user approval gate. File changes can be reviewed in VS Code's diff editor before being applied.

== Why This Shape

A full VS Code clone would require maintaining an editor distribution, extension host, update channel, and product branding. The extension-first shape validates the AI coding-agent experience quickly and leaves room for a later VSCodium/Electron distribution when requirements are clearer.
